# Cloud connections

A **cloud connection** links your Ductape workspace to an external provider. Each connection has a stable **tag** (for example `prod_aws`) that you pass as `cloud: '<tag>'` when [linking components](./cloud-linked-components).

See [Cloud overview](./overview) for the full connect â†’ link â†’ run flow.

## Lifecycle

| Step | Method |
|------|--------|
| Create | `connections.create({ provider, name, scopes })` |
| **Complete** | **`connections.complete(cloudTag, input)`** â€” save provider credentials |
| Validate | `connections.validate(cloudTag)` â€” test and activate |
| Read | `connections.fetch(cloudTag)` / `connections.list()` |
| Update | `connections.update(cloudTag, { display_name, description, scopes })` |
| Delete | `connections.delete(cloudTag)` |

Always use the connection **tag** in `complete`, `validate`, `fetch`, `update`, and `delete` â€” not an internal ID.

```ts
await ductape.cloud.connections.create({
  provider: 'aws',
  name: 'prod_aws',
  scopes: ['storage', 'database'],
});

await ductape.cloud.connections.complete('prod_aws', {
  role_arn: 'arn:aws:iam::123456789012:role/DuctapeAccess',
});

await ductape.cloud.connections.validate('prod_aws');
```
