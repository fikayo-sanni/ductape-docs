# Configuration

The Ductape MCP server uses **stdio transport** and requires **no environment variables**. Configure your MCP client to spawn the server as a child process.

The examples below assume you have installed the server globally (`npm install -g @ductape/mcp`). If you prefer not to install globally, replace `"command": "ductape-mcp"` with `"command": "npx", "args": ["-y", "@ductape/mcp"]` in any config block.

## Cursor

Add the server in MCP settings. Config file locations:

- **User-wide:** `~/.cursor/mcp.json`
- **Project-scoped:** `.cursor/mcp.json` in your project root

```json
{
  "mcpServers": {
    "ductape": {
      "command": "ductape-mcp"
    }
  }
}
```

**Using `npx` (no global install required):**

```json
{
  "mcpServers": {
    "ductape": {
      "command": "npx",
      "args": ["-y", "@ductape/mcp"]
    }
  }
}
```

After editing config, **restart Cursor** or reload MCP servers from the MCP settings panel.

## Claude Desktop

Claude Desktop uses a similar JSON config. On macOS:

```
~/Library/Application Support/Claude/claude_desktop_config.json
```

On Windows:

```
%APPDATA%\Claude\claude_desktop_config.json
```

```json
{
  "mcpServers": {
    "ductape": {
      "command": "ductape-mcp"
    }
  }
}
```

Restart Claude Desktop after saving.

## Other MCP clients

Any client that supports stdio MCP servers can use the same pattern:

| Field | Value |
|-------|-------|
| `command` | `ductape-mcp` (global install) or `npx` |
| `args` | omit for global binary; `["-y", "@ductape/mcp"]` for npx |
| Transport | stdio (default for spawned processes) |

Clients that support MCP over SSE or HTTP would need a separate adapter — the Ductape MCP server ships **stdio only**.

## Credentials

The server itself stores **no credentials**. Every tool call requires a `publishable_key` argument — all three tools (`ductape_execute`, `ductape_generate_payload`, `ductape_generate_snippet`) use the same key.

### Where to find your publishable key

Workbench → workspace settings → SDK access panel.

See [Security](./security) for why credentials are per-request rather than in server config.

## Project-scoped vs user-wide config

| Scope | Best for |
|-------|----------|
| **User-wide** (`~/.cursor/mcp.json`) | Personal development across all projects |
| **Project-scoped** (`.cursor/mcp.json`) | Team-shared config committed to the repo |

For team repos, using `npx` in the config avoids any machine-specific paths and works out of the box for everyone.
