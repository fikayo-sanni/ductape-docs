## Runtime defaults (TypeScript)

Set **`product`** and **`env`** on the [`Ductape` constructor](/sdk/runtime-defaults) only (not via `configure()`, env vars, or `product.init`). Omitted fields on `connect()` and queries inherit from the constructor or active connection.

```ts
const ductape = new Ductape({
  accessKey: process.env.DUCTAPE_ACCESS_KEY!,
  product: 'my-product',
  env: 'dev',
});
await ductape.databases.connect({ database: 'users-db' });
await ductape.api.run({
  app: 'stripe',
  action: 'create-charge',
  input: { amount: 1000, currency: 'usd' },
});
```

## NestJS

For NestJS apps, use **`@ductape/nestjs`** — modules, injectable handles, and decorators such as `@Api`, `@Database`, `@Storage`, and `@Webhook`. See the [NestJS integration guide](/nestjs/overview). NestJS examples are **TypeScript only** (not included in the multi-language SDK tabs above).

```bash
npm install @ductape/nestjs @ductape/sdk
```
